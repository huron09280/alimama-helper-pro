# 插件 UI 统一设计规范

本规范从“万能查数 - 人群对比看板”的现有实现中抽象插件统一视觉语言，用于后续统一主面板、万能查数、算法护航、组建计划、行内快捷入口和其它注入式页面。

参考依据：

- 运行态截图：`tasks/plugin-page-screenshots-2026-05-29/07-plugin-magic-report.png`、`10-plugin-magic-report-query-tab.png`、主面板、算法护航、组建计划向导截图。
- 样式实现：`src/main-assistant/magic-report.js`、`src/main-assistant/ui.js`。
- 图标规则：`docs/图标设计规范.md`。

## 设计定位

统一 UI 的目标不是做营销页，而是做一个嵌入阿里妈妈后台的高密度工作台。页面应轻、清晰、可扫描，优先服务反复操作、数据对比、状态确认和低误操作风险。

统一描述：

> 阿里妈妈多合一助手采用浅色玻璃拟态工作台风格，以柔和蓝紫品牌光感为主线，使用半透明面板、胶囊控制条、低饱和数据色和轻量阴影组织复杂信息。界面保持后台工具的信息密度，控件紧凑但留足点击热区，数据图表以可读性和可比较性优先，不做重装饰。

适用页面：

- 悬浮球、主面板、辅助显示开关。
- 万能查数弹窗、人群对比看板、快捷话术。
- 算法护航弹窗和结果浮层。
- 组建计划向导、矩阵配置、批量编辑弹窗。
- 计划行快捷入口、批量+ 菜单、下载捕获面板。

## 视觉原则

- 浅色优先：页面基底以白、浅蓝、浅灰蓝为主，避免大面积深色和高饱和背景。
- 玻璃材质：浮层、面板、图表容器优先使用半透明白底、轻边框、背景模糊和柔和阴影。
- 信息高密度：后台工具页面不做大留白展示型布局；控件、图例、状态和数据应围绕任务紧凑排列。
- 层级克制：标题、控制条、状态条、内容区最多形成 3 到 4 个清晰层级，不用多层卡片嵌套。
- 语义配色：业务色绑定指标或状态，不按行号、随机值或页面局部喜好换色。
- 动效轻量：动效用于状态反馈、增量出现和 hover 强调，持续时间短，不能影响阅读或操作。

## 布局系统

### 容器

- 主浮层和大弹窗使用 `18px` 圆角；人群看板最大化时可取消圆角贴合视口。
- 内容卡片和图表容器使用 `14px` 到 `16px` 圆角。
- 按钮、小卡、图例、tooltip 使用 `8px` 到 `12px` 圆角。
- 胶囊控件、计划信息条、图例组使用 `999px` 圆角。
- 大弹窗默认可居中或最大化；数据看板类页面优先使用全视口工作台，避免窄弹窗造成图表拥挤。

### 间距

- 弹窗头部内边距建议 `10px 20px` 到 `14px 20px`。
- 内容区外边距建议 `10px` 到 `12px`。
- 工具条组内间距建议 `8px` 到 `12px`。
- 胶囊按钮内边距建议 `5px 12px` 或 `7px 16px`。
- 数据网格单元间距建议 `12px`，图表内部间距建议 `6px` 到 `14px`。

### 页面骨架

统一页面优先采用以下骨架：

1. 标题区：品牌图标、页面描述、刷新/关闭/最大化等窗口动作。
2. 视图区：页签胶囊、当前对象信息、快捷动作或筛选摘要。
3. 状态区：加载、成功、警告、错误和重试入口。
4. 控制区：指标图例、周期开关、显示模式开关。
5. 内容区：表格、图表、配置表单或列表。

## 颜色与材质

### 基础 token

优先复用 `src/main-assistant/ui.js` 中的 `--am26-*` 变量：

```css
--am26-font: "SF Pro Display", "SF Pro Text", "PingFang SC", "Microsoft YaHei", sans-serif;
--am26-text: #1b2438;
--am26-text-soft: #505a74;
--am26-border: rgba(255, 255, 255, 0.4);
--am26-border-strong: rgba(255, 255, 255, 0.6);
--am26-surface: rgba(255, 255, 255, 0.25);
--am26-surface-strong: rgba(255, 255, 255, 0.45);
--am26-panel: linear-gradient(135deg, rgba(255, 255, 255, 0.4), rgba(255, 255, 255, 0.1));
--am26-panel-strong: linear-gradient(135deg, rgba(255, 255, 255, 0.6), rgba(255, 255, 255, 0.2));
--am26-primary: rgba(69, 84, 229, 1);
--am26-primary-strong: #1d3fcf;
--am26-success: #0ea86f;
--am26-warning: #e8a325;
--am26-danger: #ea4f4f;
--am26-shadow: 0 8px 32px 0 rgba(31, 38, 135, 0.15);
```

### 文本色

- 主标题和关键数据：`#1a2a47` 或 `#1b2438`。
- 正文和标签：`#4a5674`、`#505a74`。
- 次级说明：`#6b7280`、`#7f8ca9`。
- 空状态和弱提示：`#95a0b9`。
- 不要在同一区域混用过多灰阶；同类文本保持同一色值。

### 背景与边框

- 浮层背景：`var(--am26-panel-strong)` 配合 `backdrop-filter: blur(20px)`。
- 内容区背景：`rgba(255,255,255,0.4)` 或浅蓝白渐变。
- 数据容器背景：`linear-gradient(145deg, rgba(246,250,255,0.75), rgba(235,243,255,0.5))`。
- 卡片边框：优先白色半透明边框，低对比、轻量分隔。
- 阴影：以 `rgba(31,53,109,0.03-0.12)` 为主，避免厚重黑影。

## 字体与数字

- 全局字体使用 `--am26-font`，保证与 macOS 和中文后台页面兼容。
- 弹窗标题：`13px` 到 `15px`，`600-700`。
- 胶囊、图例、表格标签：`11px` 到 `13px`，`600`。
- tooltip 和辅助说明：`10px` 到 `12px`，`600`。
- 数字对齐场景启用 `font-variant-numeric: tabular-nums` 和 `font-feature-settings: "tnum" 1`。
- 后台页面不使用超大标题；紧凑面板内标题应匹配容器尺寸。

## 控件规范

### 顶部标题区

- 左侧使用共享线性图标或业务图标，右侧使用图标按钮承载刷新、关闭、最大化等动作。
- 标题文案保持一行，超出时省略，不挤压窗口动作。
- 图标按钮尺寸建议 `32px`，圆角 `8px`，hover 使用浅灰底；危险动作 hover 使用浅红底和危险色。

### 页签胶囊

- 页签容器使用半透明白色渐变、`20px` 圆角、内阴影和轻外阴影。
- 页签按钮默认透明，active 使用白色半透明底、深色文字和轻阴影。
- 默认视图星标等小图标可在 hover/focus/默认态出现；不要常驻占据主要视觉注意力。

### 计划信息胶囊

- 当前计划名、计划 ID、商品 ID 等上下文信息放在同一行胶囊内。
- 文案格式统一为 `计划名：...`、`计划ID：...`、`商品ID：...`。
- 长文本必须省略；商品 ID 多选或候选切换使用自定义下拉，不使用裸浏览器 select。

### 图例与开关

- 人群、周期、显示占比、显示提示都用胶囊开关表达。
- 开关内使用小圆点承载语义色，文字承载语义；不要只靠颜色。
- off 状态使用透明背景、降低透明度和去阴影；至少保留一个指标和一个周期可见。
- 图例组之间使用弱分隔符或空隙，避免全塞进单一按钮组。

### 按钮

- 主按钮使用品牌蓝紫，白字，圆角 `8px` 到 `12px`。
- 次按钮使用半透明白底或浅品牌色底。
- 重试、刷新、辅助动作保持轻量，不要抢主任务视觉。
- 文案要短，按钮内文字不能换行或溢出。

## 数据可视化

人群对比看板的图表规范可复用到其它数据页。

### 数据色

| 指标 | 色值 | 用法 |
| --- | --- | --- |
| 点击人群 | `#2f54eb` | 蓝色，流量/点击类指标 |
| 加购人群 | `#13c2c2` | 青色，兴趣/加购类指标 |
| 成交人群 | `#fa8c16` | 橙色，转化类指标 |
| 商品成交人群 | `#52c41a` | 绿色，商品级转化指标 |

业务色必须同时用于图例圆点、柱状图、提示卡和相关 insight，保持跨组件一致。

### 数据网格

- 数据看板使用“维度行 × 周期列”的网格结构。
- 首列维度标题宽度按文字自适应；数据列使用均分。
- 表头和行头 sticky，保证横纵滚动时仍能定位当前维度。
- 单元格不重复渲染冗余标题，减少视觉噪声。
- 省份、城市等标签多的维度允许横向滚动，并支持拖拽浏览。

### 柱状图

- 柱体使用同指标色的垂直渐变；顶部圆角 `6px`，底部圆角 `2px`。
- 柱宽使用 `clamp()` 限制，随可见指标数和标签密度自动收缩。
- 标签多于 7 个进入 dense，10 个以上进入 ultra dense。
- 柱组之间使用极弱竖向渐变分隔线，提高横向可读性。
- 占比值默认由全局开关控制显隐，不在局部单独覆盖。

### tooltip

- tooltip 使用浅蓝玻璃背景、白色半透明边框和轻阴影。
- 内容使用网格排版，周期、占比、pt 差异、数值列右对齐。
- 多指标对比时，第一行展示人群列头；缺失值留空，不用 `--` 占位制造噪声。
- 正向差异使用 `#0f766e`，负向差异使用 `#b42318`，中性差异使用 `#a16207`。
- 不使用原生 `title` 作为核心提示；需要即时反馈时使用受控 tooltip。

## 交互状态

### 加载

- 状态条使用单行文案，带进度条背景变量。
- 加载中允许增量渲染已完成数据，避免空白等待。
- 增量出现使用轻微上移动画：`opacity 0 -> 1`、`translateY(7px) -> 0`，时长约 `0.34s`。

### 成功、警告、错误

- 成功使用浅绿色背景和 `--am26-success` 语义。
- 警告使用浅黄色背景和 `--am26-warning` 语义。
- 错误使用浅红背景和 `--am26-danger` 语义。
- 成功态可自动隐藏；错误态保留并提供重试入口。

### hover 与 focus

- hover 以轻阴影、轻位移、背景变亮为主，不做大幅缩放。
- `focus-visible` 必须有可见轮廓，常用 `rgba(37,99,235,0.45)`。
- 图标按钮 hover 不应破坏原本定位 transform。

### 空状态

- 空状态文案短且具体，例如 `暂无数据`、`未识别到当前计划ID`。
- 空状态保持在内容区内，不弹出全局阻塞提示。
- 能恢复的错误给重试按钮；不能恢复的错误直接告诉用户需要进入计划详情或勾选计划。

## 响应式与可访问性

- 大看板在桌面优先全屏或宽屏，保证图表和表格不拥挤。
- 紧凑主面板使用两列工具按钮；按钮文案强制单行。
- 文本必须设置 `min-width: 0`、`overflow: hidden`、`text-overflow: ellipsis`，避免挤压相邻控件。
- 可交互图标需要 `aria-label` 或 `title`；装饰性图标保持 `aria-hidden`。
- 动效需兼容 `prefers-reduced-motion: reduce`。
- 涉及真实投放、创建、提交、删除的入口必须明确区分主动作和辅助动作，避免误点。

## 其它页面迁移清单

新增或重构插件页面时，按以下顺序对齐：

1. 先判断页面类型：小面板、弹窗工作台、数据看板、行内增强、菜单。
2. 复用 `--am26-*` token，不在局部重新定义一套品牌色、字体和阴影。
3. 采用统一骨架：标题区、上下文区、状态区、控制区、内容区。
4. 所有图标走 `renderAmIcon()` 或 `renderAmWindowIcon()`，遵守 `docs/图标设计规范.md`。
5. 表格和图表优先用浅玻璃容器、低对比边框和 sticky 关键头部。
6. 指标色按语义绑定，不按页面局部顺序随机分配。
7. 状态反馈必须覆盖 loading/success/warn/error/empty/retry。
8. UI 改动后用真实插件截图核对，不只看静态代码。

## 禁止项

- 禁止为每个页面单独发明一套主题色、圆角、阴影和字体。
- 禁止在后台工具页使用营销页式大标题、大卡片堆叠或装饰性背景。
- 禁止卡片套卡片造成层级过重；需要分组时优先用间距、弱边框和 sticky 标题。
- 禁止用 emoji、私有字体字符或裸文本符号冒充功能图标。
- 禁止只靠颜色表达开关或状态，必须有文字、图标或 `aria` 语义。
- 禁止使用粗黑阴影、强饱和渐变、大面积单色背景破坏原生后台融合感。
- 禁止只改 `dist/` 或根 userscript；运行态样式必须从 `src/` 源码维护并构建同步。

## 验收清单

- 是否使用 `--am26-*` 基础 token，且未新增重复主题系统？
- 弹窗、卡片、按钮圆角是否落在统一范围内？
- 文字层级是否能在截图中快速区分标题、正文、辅助信息和状态？
- 业务色是否绑定语义，并在图例、图表、tooltip 中一致？
- 数据密集页面是否可横向/纵向扫描，不因窄容器产生重叠？
- hover、focus、active、disabled、loading、empty、error 是否都有明确视觉状态？
- 图标是否符合 `docs/图标设计规范.md`，没有 emoji 或私有字体残留？
- 真实运行截图是否与 `tasks/plugin-page-screenshots-2026-05-29/` 中的主视觉保持一致？
- 若涉及 UI 或真实页面运行态，是否用 Chrome DevTools MCP 在 `one.alimama.com` 做过验证并记录？
