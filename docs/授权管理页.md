# 授权管理页（本地 HTML，对齐阿里云线上 API）

## 1. 目标与边界
- 目标：通过 `dev/license-admin.html` 查看当前授权状态、正在使用插件的店铺，并执行授权/吊销操作。
- 边界：本地 HTML 仅是管理前端；授权真值由 `services/license-server/index.mjs` 维护。
- 同步规则：线上 `GET /` 管理页模板来源于 `services/license-server/license-admin.html`，与 `dev/license-admin.html` 保持同版能力（首次使用、授权到期、日期直设）。
- 安全约束：不在前端绕过授权。插件端仍通过 `/v1/license/verify` 决定是否放行。

## 2. 线上真实接口（已按源码核对）
以下路由均来自 `services/license-server/index.mjs` 的 `handler` 分发：

- `GET /v1/license/admin/state`
- `POST /v1/license/admin/allow`
- `POST /v1/license/admin/revoke`
- `POST /v1/license/admin/delete`
- `POST /v1/license/verify`
- `POST /v1/license/revoke`（同样要求管理员鉴权）

管理页页面入口：
- `GET /`
- `GET /admin`
- `GET /index.html`

线上页面模板来源：
- `services/license-server/license-admin.html`

## 3. 管理鉴权
管理接口读取以下请求头之一：

- `x-am-admin-token`
- `x-admin-token`（兼容）

对应环境变量：
- `AM_LICENSE_ADMIN_TOKEN`

说明：
- `AM_LICENSE_ADMIN_TOKEN` 为必填；未配置时，管理接口会返回 `503 admin_token_not_configured`。
- 已配置后，必须携带正确 token。

## 4. 你关心的三个问题
### 4.1 在哪里看“现在用这个插件的店铺”？
- 看 `GET /v1/license/admin/state` 的 `activeShops`。
- `activeShops` 由每次 `/v1/license/verify` 调用写入内存聚合（包含 `lastSeenAt`、`scriptVersion`、`buildChannel`、`verifyCount` 等）。
- 管理页会直接展示：
  - 子账号（原始 `shopName` 中 `:` 后部分）
  - 店铺名称（原始 `shopName` 去掉 `:` 后子账号部分）
  - 首次使用时间（`firstSeenAt`）
  - 最近活跃时间（`lastSeenAt`）
  - 授权到期时间（`authExpiresAt`）
  - 浏览器版本（`browserVersion`）
  - 操作系统版本（`osVersion`）
- 页面默认“最近活跃（小时）”筛选值为 `0`（不过滤），避免新店铺因暂未上报 `lastSeenAt` 被默认隐藏。
- 状态筛选支持：`全部 / 已授权 / 未授权 / 已吊销 / 已过期`。
- 新增“到期筛选”：`全部 / 已过期 / 未过期`，用于快速筛查临期与过期店铺。
- 新店首次 `verify` 默认不会自动授权；如需开启自动放行，设置 `AM_LICENSE_AUTO_PROVISION_NEW_SHOP=1`。

### 4.2 怎么剔除授权、限制使用？
- 取消授权：`POST /v1/license/admin/allow`，传 `{"shopId":"...","enabled":false}`
- 重新授权（默认 3 天有效期）：`POST /v1/license/admin/allow`，传 `{"shopId":"...","enabled":true}`（服务端默认 `durationKey=3d`）
- 指定有效期：`POST /v1/license/admin/allow`，可附加 `durationKey`：
  - `3d`（3天）
  - `1m`（1个月）
  - `3m`（3个月）
  - `1y`（1年）
  - `permanent`（永久）
- 授权时长维度：管理页对“授权放行/取消授权/设置到期日期”按店铺名称维度同步（同店多子账号会一起生效），不再按单子账号割裂。
- 回填店名：`POST /v1/license/admin/allow`，传 `{"shopId":"...","enabled":<当前值>,"shopName":"店铺名"}`（管理页行级“操作下拉 -> 设置店名”选中后自动执行该调用）
- 直接设置到期日期：在列表“授权到期”列直接点击日期输入框，修改后会自动调用 `POST /v1/license/admin/allow` 写入 `authExpiresAt`（无需再走操作下拉）。
- 到期列“永久”快捷按钮：点击后会调用 `POST /v1/license/admin/allow` 并携带 `durationKey=permanent`，将授权到期置为永久。
- 吊销：`POST /v1/license/admin/revoke`，传 `{"shopId":"...","revoked":true}`
- 取消吊销：`POST /v1/license/admin/revoke`，传 `{"shopId":"...","revoked":false}`
- 删除当前行：`POST /v1/license/admin/delete`，传 `{"shopId":"..."}`（删除该店铺授权/吊销/活跃聚合项，并写入“删除重置”标记；该店铺后续再次 `verify` 会按新店铺语义重新接入）。

### 4.3 可以用一个本地 HTML 来管理吗？
- 可以，`dev/license-admin.html` 已对齐上述真实接口。
- 推荐本地静态服务访问：
  - `python3 -m http.server 8173`
  - 浏览器访问：`http://127.0.0.1:8173/dev/license-admin.html`

### 4.4 我没有购买域名，只能用默认域名，为什么会下载？
- 命中场景：访问 `bucket.oss-cn-*.aliyuncs.com/*.html` 或默认 `fcapp.run` 域名时，若响应头出现 `Content-Disposition: attachment`，浏览器会直接下载，页面通常表现为 `net::ERR_ABORTED`。
- OSS 典型特征：`x-oss-force-download: true`，常伴随 `x-oss-ec: 0048-00000001`。
- 官方依据：阿里云文档《[如何配置访问OSS文件时的预览行为？](https://help.aliyun.com/zh/oss/how-to-ensure-an-object-is-previewed-when-you-access-the-object)》说明该策略属于默认域名限制，需使用自定义域名才能保证浏览器预览。
- 端点认知修正（2026-04-07 复核）：静态网站托管并不是“保证默认域名可预览”的开关。阿里云文档《[通过 Bucket 域名访问 OSS](https://help.aliyun.com/zh/oss/user-guide/access-oss-via-bucket-domain-name)》明确静态网站仍通过 Bucket 访问域名访问；在本次复核中，`*.oss-website-<region>.aliyuncs.com` 在公共 DNS 返回 `NXDOMAIN`，不应再作为可用默认入口。
- 若默认对象域名直接返回 `403 AccessDenied`（`x-oss-ec: 0003-00000001`），通常是 Bucket ACL 回退为 `private` 导致，需要在 OSS 控制台将 Bucket ACL 改回 `public-read`（同时确认“阻止公共访问”关闭）。
- 在“无域名”前提下的可用方案（本项目推荐）：
  - 入口用本地管理页：`dev/license-admin.html`（或 `http://127.0.0.1:8173/dev/license-admin.html`）；
  - `Base URL` 仍填默认 FC 地址（例如 `https://xxx.cn-hangzhou.fcapp.run`）；
  - 管理操作继续走线上 API：`/v1/license/admin/state|allow|revoke|delete`。

### 4.5 不买域名，如何用默认域名线上打开管理页？
可以，建议把前端托管到第三方静态站点默认域名，后端继续走 FC 默认域名 API：

1. GitHub Pages
- 工作流：`.github/workflows/license-admin-pages.yml`
- 部署后地址形态：`https://<user>.github.io/<repo>/`
- 说明：工作流会自动把 `dev/license-admin.html` 发布为 Pages 首页。

2. Vercel
- 配置文件：`vercel.json`
- 部署后地址形态：`https://<project>.vercel.app/`
- 说明：根路径 `/` 已重写到 `/dev/license-admin.html`。

3. Netlify
- 配置文件：`netlify.toml`
- 部署后地址形态：`https://<project>.netlify.app/`
- 说明：根路径 `/` 已重写到 `/dev/license-admin.html`。

统一接入步骤：
- 打开托管后的前端地址；
- 将 `Base URL` 填为 FC 默认地址（例如 `https://xxx.cn-hangzhou.fcapp.run`）；
- 填管理员 token，点击“刷新状态”验证连通。

## 5. 核心请求示例
将 `<BASE_URL>` 替换为你的 FC HTTP 触发器地址，将 `<ADMIN_TOKEN>` 替换为管理 token。

### 5.1 查询管理状态
```bash
curl -sS -X GET "<BASE_URL>/v1/license/admin/state" \
  -H "x-am-admin-token: <ADMIN_TOKEN>" \
  -H "accept: application/json"
```

### 5.2 授权/取消授权
```bash
curl -sS -X POST "<BASE_URL>/v1/license/admin/allow" \
  -H "x-am-admin-token: <ADMIN_TOKEN>" \
  -H "content-type: application/json" \
  -d '{"shopId":"1234567890","enabled":true}'
```
> 默认授权有效期为 3 天（可通过服务端环境变量调整）。

```bash
curl -sS -X POST "<BASE_URL>/v1/license/admin/allow" \
  -H "x-am-admin-token: <ADMIN_TOKEN>" \
  -H "content-type: application/json" \
  -d '{"shopId":"1234567890","enabled":false}'
```

```bash
curl -sS -X POST "<BASE_URL>/v1/license/admin/allow" \
  -H "x-am-admin-token: <ADMIN_TOKEN>" \
  -H "content-type: application/json" \
  -d '{"shopId":"1234567890","enabled":true,"shopName":"示例店铺"}'
```

```bash
curl -sS -X POST "<BASE_URL>/v1/license/admin/allow" \
  -H "x-am-admin-token: <ADMIN_TOKEN>" \
  -H "content-type: application/json" \
  -d '{"shopId":"1234567890","enabled":true,"durationKey":"3m"}'
```

### 5.3 吊销/取消吊销
```bash
curl -sS -X POST "<BASE_URL>/v1/license/admin/revoke" \
  -H "x-am-admin-token: <ADMIN_TOKEN>" \
  -H "content-type: application/json" \
  -d '{"shopId":"1234567890","revoked":true}'
```

```bash
curl -sS -X POST "<BASE_URL>/v1/license/admin/revoke" \
  -H "x-am-admin-token: <ADMIN_TOKEN>" \
  -H "content-type: application/json" \
  -d '{"shopId":"1234567890","revoked":false}'
```

### 5.4 删除当前行
```bash
curl -sS -X POST "<BASE_URL>/v1/license/admin/delete" \
  -H "x-am-admin-token: <ADMIN_TOKEN>" \
  -H "content-type: application/json" \
  -d '{"shopId":"1234567890"}'
```

### 5.5 插件校验（无需管理员 token）
```bash
curl -sS -X POST "<BASE_URL>/v1/license/verify" \
  -H "content-type: application/json" \
  -d '{
    "shopId":"1234567890",
    "shopName":"示例店铺",
    "scriptVersion":"7.01",
    "buildId":"20260401",
    "buildChannel":"release",
    "runtimeMode":"userscript",
    "deviceHash":"abc123",
    "timestamp":1711958400000,
    "nonce":"n-001"
  }'
```
说明：
- `timestamp` 需落在服务端允许时间窗（默认 ±60 秒）；
- `nonce` 在防重放窗口内只能使用一次（默认 5 分钟），重复提交会返回 `nonce_replayed`。

## 6. `admin/state` 关键字段
`GET /v1/license/admin/state` 返回：

- `allowed.env/memory/effective`
- `revoked.env/memory/effective`
- `deleted.memory`（被管理台删除后处于“重置”状态的店铺）
- `shops`（env + memory 合并）
- `activeShops`（verify 聚合）
  - 包含 `browserVersion`、`osVersion`、`lastReason/lastCode` 等运行态字段
- `leaseTtlMs`
- `defaultAuthValidDays`（默认授权有效天数）
- `authHardening`（自动授权开关、时间窗、nonce 窗口、policy token 签发状态）
- `generatedAt`
- `adminTokenConfigured`

本地管理页就是读取这份结构并做筛选/分页展示。
