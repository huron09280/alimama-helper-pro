# 图标设计规范

本规范用于阿里妈妈多合一助手的运行态图标设计与实现。后续新增或修改插件图标时，优先遵守本文，再结合具体页面视觉做小范围调整。

参考方法：站酷文章《ICON设计干货来啦~》的功能性图标原则。项目落地口径是：功能表达清晰、结构简练、工整有序、同套图标保持统一画板和线性语言。

## 适用范围

- 主面板、悬浮球、工具按钮、弹窗标题和弹窗控制按钮。
- 万能查数快捷话术、刷新、关闭、默认视图星标、商品下拉箭头。
- 计划 ID 快捷入口、并发开启入口。
- 算法护航标题、居中、最大化、关闭、结果状态图标。
- 关键词向导内可交互的下拉、勾选、半选、展开收起等图标。
- Chrome extension 应用图标。

不纳入本规范强制替换范围：

- 日志内容中的状态文本符号，例如成功、失败、警告提示。
- 文档截图、mockup HTML、历史更新日志。
- 纯 CSS 装饰点、进度条、背景纹理。

## 视觉原则

- 功能优先：图标必须让用户能直接判断动作含义，不追求复杂装饰。
- 结构简练：优先使用 1 到 3 个核心形状表达，不堆叠细节。
- 风格统一：同一界面内使用同一线宽、端点、圆角和视觉重心。
- 像素友好：小尺寸下不能依赖过细线条、复杂镂空或密集转折。
- 与文字互补：图标不替代关键文案，文案也不应混入 emoji 图标。

## 基础规格

- 画板：所有 UI 功能图标使用 `viewBox="0 0 24 24"`。
- 主绘制区：控制在 20×20 内，通常从 `x=2` 到 `x=22`、`y=2` 到 `y=22`。
- 坐标：优先使用整数坐标；圆形、斜线等允许少量 `.5` 用于视觉居中。
- 线宽：默认 `stroke-width="2"`；极小尺寸可在调用处设为 `1.6` 到 `2.4`。
- 线端：统一 `stroke-linecap="round"`。
- 拐角：统一 `stroke-linejoin="round"`。
- 填充：UI 功能图标默认 `fill="none"`，仅星标激活、应用图标等少数语义可用填充。
- 颜色：默认 `currentColor`，由按钮或状态容器继承；不要在普通 UI 图标里硬编码颜色。

## 尺寸使用

- 悬浮球主品牌图标：`22px`。
- 主面板标题和工具按钮：`16px`。
- 弹窗控制按钮：`14px` 到 `18px`，按按钮尺寸适配。
- 快捷话术胶囊图标：`13px`。
- 默认星标、小型勾选、下拉箭头：`10px` 到 `14px`。
- 状态大图标：可放大到 `48px` 到 `56px`，但仍使用同一 24×24 矢量源。

## 应用图标

Chrome extension 图标与 UI 功能图标分开管理。

- 源稿：`src/entries/extension-icons/icon.svg`。
- 画板：`viewBox="0 0 128 128"`。
- 当前图形：复用悬浮球 `renderAmIcon('logo')` 的闪电路径，等比换算到 128 画板。
- 禁止项：不要叠加圆形底板、外圈、圆角矩形底板或额外装饰线。
- PNG 输出：`icon-16.png`、`icon-32.png`、`icon-48.png`、`icon-128.png`。
- 生成方式：用 `sharp` 从 `icon.svg` 生成，不手工修改 `dist/extension`。

## 代码实现规范

UI 功能图标统一走共享渲染入口：

```js
renderAmIcon('shield-check', { size: 16 })
```

共享定义位置：

```text
src/shared/script-preamble.js
```

新增图标时：

- 在 `AM_ICON_DEFS` 增加 kebab-case 名称，例如 `shield-check`、`chevron-down`。
- `body` 只写 24×24 内的 SVG 子元素，不写完整 `<svg>`。
- 优先使用 `<path>`、`<circle>`、`<rect>` 等基础元素。
- 默认复用 `renderAmIcon()` 的 `fill`、`stroke`、`stroke-width`、端点和拐角配置。
- 若必须填充，单个图标定义中显式写 `fill` 与 `stroke: 'none'`。

调用侧要求：

- 装饰性图标默认不传 `label`，保持 `aria-hidden="true"`。
- 独立表达动作的按钮必须有 `title` 或 `aria-label`。
- 不在按钮文案里混入 emoji；图标和文字要拆成 SVG + 文本。
- 不直接复制第三方 iconfont 的 `viewBox="0 0 1024 1024"` 大路径。
- 不使用私有字体字符作为图标。
- CSS data URI 图标只允许用于难以插入 DOM 的样式控件，并且必须保持 24×24 线性 SVG 语言。

## 色彩规范

- 默认态：继承当前文本色，例如 `var(--am26-text-soft)`。
- Hover/active：由按钮状态修改 `color`，图标继续继承 `currentColor`。
- 成功：`var(--am26-success,#0ea86f)`。
- 警告：`var(--am26-warning,#e8a325)`。
- 危险/关闭：`var(--am26-danger,#ea4f4f)`。
- 品牌主色：`var(--am26-primary,#2a5bff)` 或应用图标底板 `#4554E5`。

## 禁止项

- 禁止在运行态 UI 新增 emoji 作为功能图标。
- 禁止新增 1024 iconfont 大路径内联 SVG。
- 禁止同一按钮组混用线性图标、填充图标、emoji 和字体图标。
- 禁止在普通 UI 图标里硬编码品牌色，导致 hover/disabled 状态失效。
- 禁止只改 `dist/` 或根 userscript；必须改 `src/` 后通过构建同步。

## 新增或修改图标流程

1. 先确认能否复用 `AM_ICON_DEFS` 里的现有图标。
2. 不能复用时，在 `src/shared/script-preamble.js` 新增图标定义。
3. 用 `renderAmIcon()` 替换调用处，不复制完整 SVG。
4. 若涉及 extension 应用图标，先改 `icon.svg`，再生成 PNG。
5. 更新或补充回归测试，至少防止退回 emoji 或 `1024` iconfont。
6. 运行构建和校验。
7. 涉及真实页面 UI 时，用 Chrome DevTools MCP 在 `one.alimama.com` 验证。

## 验收清单

- 图标 `viewBox` 是否为 `0 0 24 24`，应用图标除外。
- 小尺寸下是否仍能识别核心语义。
- 同一按钮组线宽、圆角、端点是否一致。
- 图标是否继承 `currentColor`，hover/disabled 是否正常。
- 文案是否未混入 emoji 图标。
- `dist/extension`、`dist/packages`、根 userscript 是否由构建生成。
- `node scripts/build.mjs --check`、相关测试和 `npm run review` 是否通过。
