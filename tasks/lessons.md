# Lessons

## 2026-03-25 用户修正：省份/城市排序默认要按各周期，主周期优先改为可切换
- 问题：如果默认直接按 90/30/7/3 稳定排序，会改变用户当前“按各周期看本期排序”的阅读习惯。
- 纠偏规则：省份/城市默认使用各周期独立标签顺序；仅在用户点击排序图标时切换为主周期优先（90→30→7→3），并支持再次点击回退默认。

## 2026-03-25 用户修正：省份/城市稳定排序要以90天为主周期
- 问题：仅按全周期累计值排序时，鼠标对比和阅读顺序不一定体现“90天同期”优先，容易偏离业务主视角。
- 纠偏规则：动态标签维度（省份/城市）在已对齐同标签前提下，稳定排序必须先比较 `90天`，缺失再依次比较 `30/7/3天`，最后才回退累计值和初始顺序。

## 2026-03-25 用户修正：省份/城市跨周期比较必须先做标签对齐
- 问题：若每个周期按自身返回顺序渲染，跨周期对比会退化成“第 N 个对第 N 个”，省份/城市会错位。
- 纠偏规则：对动态标签维度（省份/城市）必须先构造“全周期稳定标签顺序”，各周期按统一标签列表回填缺失值后再比较。

## 2026-03-25 用户修正：明确“只改下拉弹窗，不改触发器”
- 问题：用户说“下拉弹窗样式”时，若同时改触发器，容易偏离需求边界。
- 纠偏规则：UI样式需求出现“弹窗/面板/下拉层”等明确词时，优先限制改动范围到对应层级，避免顺手修改触发器或其他关联控件。

## 2026-03-25 用户修正：下拉视觉需对齐“周期置顶表头”样式
- 问题：自定义下拉虽可用，但视觉与看板内部“周期置顶表头”风格不一致，整体感割裂。
- 纠偏规则：同一看板内新增控件需优先复用既有主视觉语义（白玻璃背景、轻边框、blur、阴影），避免出现独立风格分叉。

## 2026-03-25 用户修正：自定义下拉必须防止继承 nowrap 导致横排
- 问题：头部容器使用 `white-space: nowrap` 时，自定义下拉若不重置容器换行且选项未强制块级，会出现“选项横向一排”。
- 纠偏规则：自定义下拉组件必须显式加两条兜底样式：下拉容器 `white-space: normal`，选项节点 `display: block`。

## 2026-03-25 用户修正：商品下拉必须改为网页自定义组件，不能用系统下拉
- 问题：即使去掉边框，原生 `<select>` 的展开层仍由系统/浏览器接管，样式与站内 UI 无法统一。
- 纠偏规则：涉及“统一网页风格”的下拉交互时，必须使用自定义触发器 + 自定义选项面板（HTML/CSS/JS），禁止继续依赖原生下拉弹层。
- 交互规则：自定义下拉至少支持三件事：点击触发器展开/收起、点击选项触发业务更新、点击外部或 `Esc` 关闭弹层。

## 2026-03-24 用户修正：局部刷新必须先替换旧指标缓存
- 问题：`itemdeal` 局部刷新若直接 merge 成功结果，失败周期会残留旧商品数据，导致切换后“新旧商品混展示”。
- 纠偏规则：局部刷新必须先按 `metric|period` 清空该指标历史结果，再写入本次成功结果；失败周期要显式表现为该指标无数据。

## 2026-03-24 用户修正：刷新中切换不能吞掉后续请求
- 问题：刷新进行中直接 `return` 会导致用户后续切换丢失，UI 选中值与图表数据可能不一致。
- 纠偏规则：刷新中收到新切换时只保留“最新一次”排队请求，当前刷新结束后立即补跑。

## 2026-03-24 用户修正：状态未知商品不能因为“有推广中项”被整体过滤
- 问题：候选过滤若在存在 `active===true` 时丢弃 `active===null`，会误删状态信息缺失但仍可用的商品。
- 纠偏规则：候选过滤只剔除 `active===false`；排序层面再让 `active===true` 优先，兼顾可用性与默认选择稳定性。

## 2026-03-24 用户修正：下拉为空（状态字段误判）
- 问题：不同接口里的 `status` 数值语义不统一，直接按 `0/1` 映射开关会把正常商品误判为暂停，导致下拉被过滤为空。
- 纠偏规则：商品“是否推广中”优先使用 `onlineStatus/isOnline/online` 数值字段；`status/itemStatus` 等仅用于文本关键词判定，禁止直接数值判停。
- 兜底规则：若状态来源不确定，默认 `active = null`（未知）而非 `false`，避免误杀可用商品。

## 2026-03-24 用户修正：商品下拉必须展示标题且过滤暂停，切换只刷新 itemdeal
- 问题：仅展示商品ID会增加识别成本；且把暂停推广商品展示在下拉里会导致“选了不可投商品”。
- 纠偏规则：商品下拉文案必须包含 `商品标题 + 商品ID`，并在候选聚合阶段优先保留 `active === true`，显式暂停(`active === false`)不展示。
- 刷新规则：商品切换事件禁止触发人群看板全量重跑；必须仅刷新 `itemdeal` 并与既有指标结果合并渲染。
- 工程规则：涉及“局部刷新”需求时，先沉淀结果缓存（按 `metric + period`）再增量更新，避免因全量重查导致性能和可用性回退。

## 2026-03-24 用户修正：计划商品下拉必须“全量商品 + 花费降序首项”
- 问题：只基于宽泛递归扫描提取商品花费会混入非商品级指标，导致默认商品不是花费最高；且若不走计划详情+单元详情补齐，会漏掉计划内商品ID。
- 纠偏规则：商品花费排序必须基于结构化报表行（计划/单元列表）提取；禁止把任意对象层级 `charge` 直接累计到商品。
- 全量规则：商品ID候选必须同时合并三路：报表花费提取、计划详情候选、单元详情候选；禁止在“首个命中”后提前停止全量补齐。
- 默认选择规则：下拉默认值按 `spend DESC, itemId ASC` 取第一项；用户手动切换后查询必须锁定所选商品，避免自动漂移。

## 2026-03-24 用户修正：并发数不应按场景硬编码降级
- 问题：并发功能若对某个场景（如全站）硬编码为固定并发，会与用户对“并发数配置全局生效”的预期冲突。
- 纠偏规则：提交并发数统一由用户配置驱动，禁止在场景层做隐式覆盖；如需限制必须显式告知并可配置。
- 汇总规则：同计划在同次运行中若已成功，重复提交类失败（如“请勿重复提交”）应在最终汇总中去重，避免误导失败统计。

## 2026-03-24 用户修正：并发语义必须是“全计划同时 + 每计划按并发数复制”
- 问题：并发复制逻辑若只覆盖“单计划”场景，会导致多计划批次与用户对并发语义的理解不一致。
- 纠偏规则：并发参数语义统一为“批次内每个计划都按并发数复制提交”，不能绑定 `remainingEntries.length === 1` 这类单计划条件。
- 验证规则：并发相关改动必须覆盖三类断言：并发分支开关、`Promise.all` 全计划并行、单计划复制提交调用。

## 2026-03-24 用户修正：立即投放默认提交方式必须是单条
- 问题：批量建计划的“立即投放”默认提交方式若是并发，会与用户预期不一致并放大失败排查成本。
- 纠偏规则：涉及“默认提交方式”的需求，必须同时修改三层：草稿默认值、UI 空值兜底、执行链路空值兜底，禁止只改其中一处。
- 测试规则：同步更新源码契约测试（默认值 + 归一化兜底），并在构建产物上执行定向回归，防止 src 与根脚本行为不一致。

## 2026-03-23 用户修正：每轮修复都要继续 3+ AGENTS 并行
- 问题：连续“继续”修复场景里，如果只做单线程修复，容易遗漏并行排查与风险复核。
- 纠偏规则：每完成一个修复点后，立即继续建立不少于 3 个 AGENTS 分工（失败根因、最小补丁、回归风险）并行推进。
- 输出规则：阶段更新中明确写出“继续建立不少于 3 个 AGENTS 去高效修复”，形成可核对执行痕迹。

## 2026-03-23 测试提取边界不可绑定远端 marker
- 问题：`mergeGoalWarnings` 测试用 `const validate =` 作为截取终点，helper 重构插入后导致函数源码切片污染并触发语法错误。
- 纠偏规则：源码切片测试优先使用“紧邻下一个定义”作为终点，避免跨越多段 helper；必要时加双 marker 兜底。
- 验证规则：切片函数执行前先验证 `typeof fn === 'function'`，并覆盖一个边界输入，防止仅靠静态正则误报。

## 2026-03-23 早退分支必须保持结果结构一致
- 问题：`createPlansBatch` 多个早退分支只返回最小计数字段，缺少 runtime/策略开关/submitEndpoint 等上下文，导致排障信息不对齐。
- 纠偏规则：在业务允许的前提下，早退分支统一返回公共字段：`runtime`、`marketingGoal`、`goalWarnings`、`submitEndpoint`、`fallbackPolicy/conflictPolicy/stopScope`、`strictGoalMatch/allowFuzzyGoalMatch`、`rawResponses`。
- 测试规则：为每类早退路径保留独立契约测试（本轮为 `keyword-create-early-return-shape.test.mjs`），避免仅靠单分支测试覆盖。
- failure 规则：`failures[*]` 默认维持统一五字段（`planName/item/marketingGoal/submitEndpoint/error`），禁止仅返回 `error`。

## 2026-03-22 用户修正：万能查数头部层级与视觉收敛
- 问题：首次实现把 `am-magic-view-tabs` 提到 header 最顶层，未满足用户对 `am-magic-header-main` 顶部优先级的明确要求。
- 纠偏规则：当用户给出精确选择器级别的布局要求时，优先按该 DOM 层级执行，不做主观“更优”重排。
- 视觉规则：分段控件改版时先控制字号与体积，默认优先 12-13px，再根据反馈放大，避免一次性改到 15px+。
- 风格规则：若页面已有玻璃/半透明语义，新增灰色容器优先使用半透明渐变 + 轻边框 + 内阴影，避免实色块割裂。

## 2026-03-22 用户修正：跨区域搬迁信息条到 header
- 问题：将 `am-crowd-matrix-campaign` 位置迁移后，若不补视图显隐控制，会在 query 视图显示无关信息。
- 纠偏规则：任何“从内容区搬到头部/全局区”的节点，必须同步梳理 `switch*View` 显隐逻辑，而不只改 DOM/CSS。
- 布局规则：header 左右分布信息时，统一使用同一视觉语义（同类渐变、边框、阴影），并为右侧长文本加 `ellipsis` 兜底。

## 2026-03-22 用户修正：右侧 vs 紧接语义
- 问题：用户说“放右边”时，后续又明确“不是右侧，而是接着”；说明需求是“紧邻”而非“右对齐”。
- 纠偏规则：当文案含糊出现“右边”时，优先确认是“右对齐”还是“右侧紧邻”；若无确认，先采用紧邻布局，避免 `margin-left: auto`。

## 2026-03-22 用户修正：插件滚动穿透页面
- 问题：插件内滚动容器触底后，wheel 事件继续传递导致原网页滚动，影响操作稳定性。
- 纠偏规则：对跨页面悬浮插件，默认同时加两层保护：`overscroll-behavior: contain` + wheel 捕获守卫兜底。
- 验证规则：必须在真实页面复现“滚到底继续滚”场景并确认页面不再位移，而不是只看代码。

## 2026-03-22 用户修正：按指定 selector 精确对齐样式
- 问题：当用户指定“按某个具体按钮样式”时，只做近似会被判定不符合预期。
- 纠偏规则：优先抓取目标 selector 的计算样式，再逐项对齐；至少覆盖 `background/border/radius/font/padding/shadow/display/gap`。
- 注意事项：若视图切换逻辑写了内联 `style.display`，会覆盖 CSS `display`，需要同步修正。

## 2026-03-22 用户修正：背景需与 corner 同源 + 元素下对齐
- 问题：用户要求 tabs 背景必须与 `am-crowd-matrix-corner` 完全同源，且 campaign 与 tabs 需要下边缘对齐。
- 纠偏规则：遇到“用下面这个一样的背景设置”时，直接复用目标渐变值，不做近似色替换。
- 布局规则：同一行组件若被要求“下对齐”，优先在父容器使用 `align-items: flex-end`，并用实页几何值（如 `bottom`）验证。
